# Use Case: Delete a Todo

## 1. Overview
- **Use Case ID:** UC-005
- **Primary Actor:** End User (browser)
- **Description:** The user permanently removes a todo item. The item is deleted from the database and disappears from the UI list.
- **Level:** User goal

## 2. Preconditions
- The TodoApp server is running and reachable.
- At least one todo item exists in the database and is visible in the UI.

## 3. Trigger
The user clicks the "Delete" (or trash-icon) control on a todo item.

## 4. Main Success Scenario
1. The user clicks the delete control on a todo item.
2. The UI sends `DELETE /todos/{id}`.
3. The API looks up the record by `id` in the repository.
4. The repository executes a DELETE statement for the record.
5. The API responds `204 No Content`.
6. The UI removes the item from the rendered list.

## 5. Alternate Flows

**5a. Todo not found**
- At step 3, if no record matches `id`, the API returns `404 Not Found`.
- The UI displays an error ("Item not found") and refreshes the list to reflect current state.

**5b. Server unreachable**
- At step 2, the fetch call fails; the UI displays a network error and the item remains in the list.

**5c. User confirmation step (optional UI pattern)**
- Before step 2, the UI may present a confirmation prompt ("Are you sure?").
- If the user cancels, no request is sent and the item remains unchanged.

## 6. Postconditions
- **Success:** The todo record no longer exists in the database. The item is absent from the UI list.
- **Failure:** The database record is unchanged; the UI shows an appropriate error.
