# Use Case: Add a Todo

## 1. Overview
- **Use Case ID:** UC-001
- **Primary Actor:** End User (browser)
- **Description:** The user creates a new todo item by providing a title and an optional description. The item is persisted and immediately appears in the todo list.
- **Level:** User goal

## 2. Preconditions
- The TodoApp server is running and reachable in the browser.
- The web UI has loaded successfully.

## 3. Trigger
The user fills in the add-todo form (title, optional description) and submits it.

## 4. Main Success Scenario
1. The user types a title (1-255 characters) in the title field.
2. The user optionally types a description (up to 2000 characters).
3. The user submits the form.
4. The UI sends `POST /todos` with JSON body `{title, description?}`.
5. The API layer validates the request via zod (title non-empty, within length limits).
6. The service assigns a unique `id`, sets `completed=false`, and records `created_at`/`updated_at` timestamps.
7. The repository executes an INSERT and returns the newly created todo object.
8. The API responds `201 Created` with the full todo JSON.
9. The UI prepends the new item to the displayed list.

## 5. Alternate Flows

**5a. Empty title submitted**
- At step 3, if the title is empty or whitespace-only, the API returns `400 Bad Request` with `{error, details}` naming the `title` field.
- The UI displays the validation error message; the list is unchanged. (See UC-006 for full validation detail.)

**5b. Title exceeds 255 characters**
- Same as 5a; `400` is returned and the UI shows an appropriate error.

**5c. Malformed JSON body**
- At step 4, if the request body is not valid JSON, the API returns `400 Bad Request`.
- The UI displays a generic error message.

**5d. Server unreachable**
- At step 4, the fetch call fails; the UI displays a network error message and the list is unchanged.

## 6. Postconditions
- **Success:** A new todo record exists in the SQLite database with the supplied title/description, `completed=false`, and server-assigned `id` and timestamps. The item is visible in the UI list.
- **Failure:** No record is created; the database state is unchanged.
